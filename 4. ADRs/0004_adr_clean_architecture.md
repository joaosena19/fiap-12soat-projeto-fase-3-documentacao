# ADR 0004 - Clean Architecture

## Status

Aceito

## Contexto

É preciso definir a estrutura interna do código. No Tech Challenge da fase 2, o uso de Clean Architecture era um requisito obrigatório para avaliação. Agora na fase 3, preciso decidir se mantenho esse padrão ou se migro para algo diferente.

## Discussão e possibilidades

Como a Clean Architecture já foi implementada na fase anterior, avaliei se valia a pena continuar com ela.

A conclusão foi que o padrão funcionou muito bem, deixando o código organizado e fácil de entender. Mudar agora seria um retrabalho desnecessário e perderia os benefícios de organização já conquistados.

Não foram consideradas outras arquiteturas pois a atual atende perfeitamente.

## Decisão

Foi decidido manter a Clean Architecture iniciada na fase 2.

## Consequências

**Positivas:**

* Código organizado e padronizado.
* Facilidade para testar regras de negócio.
* Evita retrabalho desnecessário.

**Negativas:**

* Nenhum ponto negativo identificado para o contexto deste projeto.
